# Анализ сценария 3: Полный бэкап /data/data/com.termux vs UI бэкап

**Дата:** 2026-07-10  
**Устройство:** POCO F5 / KernelSU root  
**Полный ручной бэкап:** `tar --numeric-owner -p -czf ... -C /data/data com.termux`  
**Результат:** 1.4G, 111152 записей, 1786 симлинков, `tar -tzf exit 0`  

---

## 1. Текущая реализация UI Backup (`TermuxBackupUtils.java`)

**Команда:**
```
tar --no-dereference -czpf - --numeric-owner -C /data/data/com.termux/files .
```

**Что бэкапится ($FILES = `/data/data/com.termux/files`):**
- `files/usr/`  = `$PREFIX` (установленные пакеты, бинарники, библиотеки)
- `files/home/` = `$HOME` (пользовательские данные: `.bashrc`, `.ssh`, скрипты, проекты)
- **Оба** — текущая реализация уже архивирует весь каталог `files/`, а не только `$PREFIX`

**Что НЕ бэкапится (вне `files/`):**
| Компонент | Путь | Размер | Важность |
|-----------|------|--------|----------|
| `shared_prefs/` | `/data/data/com.termux/shared_prefs/` | <1 MB | ★★ (настройки UI) |
| `databases/` | `/data/data/com.termux/databases/` | <1 MB | ★ (обычно пусто) |
| `cache/` | `/data/data/com.termux/cache/` | 50–200 MB | ☆ (временные файлы) |
| `code_cache/` | `/data/data/com.termux/code_cache/` | <1 MB | ☆ (кэш Android) |
| `lib/` | `/data/data/com.termux/lib/` | 50–100 MB | ☆ (дублирует `$PREFIX/lib`) |

## 2. Анализ: что добавляет расширение до `/data/data/com.termux`

### 2.1 shared_prefs — НИЗКИЙ РИСК, низкая ценность

Содержимое (из `TermuxAppSharedPreferences.java`):
- `show_terminal_toolbar` (boolean)
- `soft_keyboard_enabled` (boolean)
- `terminal_font_size` (int)
- `current_session` (string)
- `log_level` (int)
- `last_notification_id` (int)

**Вывод:** Только UI-настройки. **Нет OAuth-токенов, API-ключей, паролей.** Termux не хранит аутентификацию в shared_prefs. Полезно при миграции на новое устройство, но не критично.

### 2.2 databases — НИЗКАЯ ЦЕННОСТЬ

Обычно пусто или содержит незначительные кэши. Не используется для хранения данных пользователя.

### 2.3 cache + code_cache — ВРЕДНО включать

- Временные файлы пакетного менеджера (apt cache)
- Кэш загрузок
- Восстанавливаются автоматически
- Добавляют 50–200 MB мусора в архив
- **Включение увеличивает размер без пользы**

### 2.4 lib — ИЗБЫТОЧНО

- Извлечённые нативные `.so` файлы из APK
- Уже присутствуют в `$PREFIX/lib/`
- **Дублирование данных в архиве**

### 2.5 Симлинки — КОРРЕКТНО ОБРАБОТАНЫ

```bash
~/storage/shared → /storage/emulated/0/
```

Флаг `--no-dereference` в UI и поведение GNU tar по умолчанию в ручном режиме: симлинки сохраняются как тип `l`, данные внешнего хранилища **не захватываются**. Подтверждено: 1786 симлинков в полном архиве имеют тип `l`.

## 3. Ключевое ограничение: ROOT

Для чтения `/data/data/com.termux/shared_prefs/`, `/data/data/com.termux/databases/` и т.д. из приложения требуется root (KernelSU/Magisk). Android sandbox не позволяет приложению читать собственную директорию на уровне выше `files/` (хотя технически `context.getFilesDir().getParentFile()` может быть доступен для некоторых компонентов, полный доступ к `shared_prefs/` требует либо root, либо `Context.MODE_WORLD_READABLE` — что устарело и не используется).

**Без root расширенный бэкап невозможен.**

## 4. Сравнение: UI backup ($FILES) vs Полный (/data/data)

| Критерий | UI backup (`files/`) | Полный (`/data/data`) |
|----------|---------------------|----------------------|
| Установленные пакеты ($PREFIX) | ✅ | ✅ |
| $HOME (пользовательские данные) | ✅ **уже есть** | ✅ |
| shared_prefs (настройки UI) | ❌ | ✅ (но низкая ценность) |
| databases | ❌ | ✅ (но почти пусто) |
| cache (временное) | ❌ | ✅ (но **вредно**, мусор) |
| lib (native .so) | ❌ | ✅ (но дублирует $PREFIX) |
| **Размер архива** | ~900 MB – 1.1 GB | ~1.4 GB |
| **Требует root** | ❌ Нет | ✅ Да |
| **Безопасность** | ⚠️ Содержит $HOME (SSH-ключи!) | ⚠️ То же самое |
| **Восстановление** | Через SAF (выбор файла) | Только ADB + root |

## 5. Переоценка рисков с учётом актуального кода

### Важное уточнение

В отличие от первоначального описания в старой документации, текущий `TermuxBackupUtils.java` **(уже includes `$HOME`)**. Команда:
```java
new String[]{TAR_BINARY, "--no-dereference", "-czpf", "-", "--numeric-owner",
    "-C", filesDir, "."}
```
где `filesDir = "/data/data/com.termux/files"` — архивируется **весь** `files/`, включая `home/`.

Это означает, что SSH-ключи (`~/.ssh/id_rsa`), GPG-ключи и прочие секреты в `$HOME` **уже** попадают в UI-бэкап. Разницы с полным бэкапом по этому риску нет.

### 5.1 Риск shared_prefs

- **Нет токенов** — только UI-преференсы
- Даже при попадании архива в чужие руки — информация бесполезна
- **Вывод:** риск отсутствует

### 5.2 Риск $HOME (уже присутствует в UI backup)

- SSH-ключи, `.git-credentials`, `.netrc`, API-токены в скриптах
- Пользователь **контролирует** эти данные (сознательно хранит их в `$HOME`)
- Если пользователь запускает бэкап — он уже принимает ответственность
- **Вывод:** существующий риск, не усугубляется расширением

### 5.3 Сравнение рисков

| Компонент | Риск утечки | Риск при restore | Комментарий |
|-----------|-------------|------------------|-------------|
| shared_prefs | 🟢 Нулевой | 🟢 Низкий | Только UI настройки |
| $HOME | 🟡 Средний (SSH-ключи!) | 🟢 Уже бэкапится | Пользователь контролирует |
| cache | 🟢 Нулевой | 🟢 Восстанавливается | Включать бессмысленно |
| lib | 🟢 Нулевой | 🟢 Дублируется | Включать бессмысленно |
| databases | 🟢 Нулевой | 🟢 Минимальный | Почти пусто |

## 6. РЕКОМЕНДАЦИЯ

### Итоговый вердикт: **ОСТАВИТЬ КАК ЕСТЬ**

**Не добавлять опцию "Full (with app data)" в UI.** Причины:

1. **Текущий UI backup уже адекватен.** `files/` (UI backup) содержит всё действительно важное:
   - Установленные пакеты ($PREFIX)
   - Пользовательские данные ($HOME, включая SSH-ключи, конфиги, скрипты)
   - Это ~90% ценности полного бэкапа

2. **Что не бэкапится — не нужно:**
   - `shared_prefs` — низкая ценность (только UI-настройки), не содержит токенов
   - `cache` / `code_cache` — временные данные, восстановятся автоматически
   - `lib` — дублирует `$PREFIX/lib/`
   - `databases` — обычно пусто

3. **Требует root.** ~95% пользователей Termux не имеют root. Опция, доступная 5%, только запутает остальных.

4. **Раздувание архива.** Добавление cache/lib увеличит архив на 100–300 MB без реальной пользы.

5. **Усложнение restore.** Восстановление `shared_prefs` требует перезапуска приложения или корректного перечитывания PreferencesManager. При несовпадении версий возможны конфликты.

6. **Ложное чувство безопасности.** "Full backup" может создать впечатление, что остальные методы неполноценны, хотя на практике разница минимальна.

### Альтернатива: документировать инструкцию

Вместо опции в UI — добавить в документацию или раздел "About" / Help:

```
💡 Advanced backup (with app data, requires root):

# Backup:
adb shell su -c 'tar --numeric-owner -p -czf /sdcard/termux-full.tar.gz -C /data/data com.termux'

# Restore:
adb shell su -c 'am force-stop com.termux'
adb shell su -c 'rm -rf /data/data/com.termux/files && mkdir -p /data/data/com.termux/files'
adb shell su -c 'tar -xzpf /sdcard/termux-full.tar.gz --numeric-owner -C /data/data com.termux'
adb shell su -c 'restorecon -R -F /data/data/com.termux/files 2>/dev/null || true'
```

⚠️ Предупреждение в инструкции:
- Содержит SSH-ключи, GPG-ключи и все файлы из `$HOME`
- Храните архив безопасно
- После restore нужен `restorecon` для SELinux context
